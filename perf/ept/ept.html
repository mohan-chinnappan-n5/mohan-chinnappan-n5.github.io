What is Experienced Page Time (EPT)?

Experienced Page Time (EPT) is a performance metric used in Salesforce Lightning to measure the time taken for a page to load into an interactive state.

Key Aspects of EPT

  • Definition: EPT measures how long it takes for a page to reach a state where the user can meaningfully interact with it.
  • Measurement Challenge: Due to the progressive loading in Lightning, determining when a page finishes loading isn't straightforward.

Factors Affecting EPT

EPT is influenced by both client-side and server-side factors:

Client-Side Factors Server-Side Factors
Browser performance Apex processing
Hardware capabilities API call handling
Network latency XHR responses
Org complexity and custom components Server-side errors and caching

Measuring EPT

EPT is calculated from the page start until there is no more activity for two frames (~33 ms). The table below provides an example of EPT calculation:

Action Duration
Initial page request 150 ms
First component loaded 350 ms
Additional component requested by the first 400 ms
Final XHR request completed 700 ms
Two idle frames (no activity) 33 ms
Total EPT 1,633 ms

Best Practices to Improve EPT

  • Optimize components to minimize XHR requests.
  • Reduce the number of custom fields and unnecessary components on a page.
  • Cache frequently used data to avoid repetitive server-side processing.
  • Minimize client-side JavaScript work during page load.
