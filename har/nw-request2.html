Request Timing Analysis

Request Timing Breakdown

Stage Description
Blocked Time spent in the browser's queue before the request could start.
DNS Time taken to resolve the domain name into an IP address. A value of -1 means this step was skipped (due to DNS caching).
SSL Time taken to establish a secure connection (SSL/TLS handshake). A value of -1 indicates no SSL was required (e.g., HTTP request).
Connect Time to establish a connection to the server. A value of -1 suggests a persistent connection was reused.
Send Time taken to send the request data to the server.
Wait Server's processing time (from receiving the request to starting the response).
Receive Time to download the response data from the server.
