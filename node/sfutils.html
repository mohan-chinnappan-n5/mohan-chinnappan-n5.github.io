SF Utils (Node.js)

A command-line tool for Salesforce developers to execute SOQL queries, track metadata changes, run anonymous Apex, and make REST API calls with ease.

Get Started

SOQL Query Execution

How to Use

Follow these steps to get started with the SF Utils Node.js application:

üîπ Step 1: Clone the Repository

Clone the SF Utils Node.js repository from GitHub and navigate to the project directory:

git clone https://github.com/mohan-chinnappan-n5/sf-utils-node.git
cd sf-utils-node

üîπ Step 2: Install Dependencies

Install the required Node.js dependencies using npm:

npm install jsforce axios csv-writer chalk@4.1.2 punycode

üîπ Step 3: Authenticate with Salesforce

Ensure you have the Salesforce CLI installed and authenticated with your Salesforce org. Run the script and provide your Salesforce username when prompted (the last username will be remembered as a default):

node index.js

You'll be prompted to enter your Salesforce username:

[Green] Enter your Salesforce username (default: user@example.org):

Sample Authentication Screen

üîπ Step 4: Run a SOQL Query

Select option 1 from the menu, enter your SOQL query, and choose to export the results to CSV if desired.

[Green] Select an option (1-5): 1
[Green] Enter your SOQL query (e.g., SELECT Id, Name FROM Account LIMIT 10): SELECT Id, Name FROM Account LIMIT 5

Sample Query Session Screens

üîπ Step 5: Track Metadata Changes

Select option 4 from the menu to track metadata changes, fetch changes, and generate a `package.xml` file. This will work for Sandboxes and Scratch orgs where Change Tracking is enabled. Not on Production!

[Green] Select an option (1-5): 4
[Green] Enter the date to track changes since (YYYY-MM-DD, e.g., 2025-04-01), or press Enter for all changes: 2025-04-01
Then run this command in your SFDX project:
sf project retrieve start -x package.xml -o username

Sample Session Screens


Commit the changes we got into git via Pull Request to trigger the pipeline

üîπ Step 6: Execute Apex or Make REST Calls

Use option 2 to run anonymous Apex code, or option 3 to make REST API calls (GET, POST, PATCH, DELETE) with JSON and CSV export options.
