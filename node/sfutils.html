SF Utils (Node.js)

A command-line tool for Salesforce developers to execute SOQL queries, track metadata changes, run anonymous Apex, and make REST API calls with ease.

How to Use

Follow these steps to get started with the SF Utils Node.js application:

üîπ Step 1: Clone the Repository

Clone the SF Utils Node.js repository from GitHub and navigate to the project directory:

git clone https://github.com/mohan-chinnappan-n5/sf-utils-node.git
cd sf-utils-node

üîπ Step 2: Install Dependencies

Make sure have nodejs installed in your machine. Install the required Node.js dependencies using npm:

npm install

üîπ Step 3: Authenticate with Salesforce

Ensure you have the Salesforce CLI installed and authenticated (sf force auth web login [-r https://test.salesforce.com] ) with your Salesforce org. Run the script and provide your Salesforce username when prompted (the last username will be remembered as a default):

node index.js

You'll be prompted to enter your Salesforce username:

üîπ Step 4: Run a SOQL Query

Select option 1 from the menu, enter your SOQL query, and choose to export the results to CSV if desired.

üîπ Step 5: Track Metadata Changes

Select option 4 from the menu to track metadata changes, fetch changes, and generate a `package.xml` file. This will work for Sandboxes and Scratch orgs where Change Tracking is enabled. Not on Production!
