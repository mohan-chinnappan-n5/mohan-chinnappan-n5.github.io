A powerful command-line tool for Salesforce developers to execute SOQL queries, make REST and Bulk API calls, track metadata changes, and run anonymous Apex with ease.

How to Use

Follow these steps to set up and use the SFU CLI for your Salesforce development tasks.

üîπ Step 1: Clone the Repository

Clone the SFU CLI repository from GitHub and navigate to the project directory:

git clone https://github.com/mohan-chinnappan-n5/sf-utils-node.git
cd sf-utils-node/2

üîπ Step 2: Install Dependencies

Ensure you have Node.js installed. Then, install the required dependencies:


npm install
chmod +x sfu.js
ln sfu.js sfu

üîπ Step 3: Authenticate with Salesforce

Ensure you have the Salesforce CLI installed and authenticated with your org using:

sf force auth web login [-r https://test.salesforce.com]

The SFU CLI uses these credentials and stores the last-used username in last_username.txt for convenience.

üîπ Step 4: Run Commands

Use the following commands to interact with your Salesforce org. All commands require the -u option unless last_username.txt is set.

SOQL Query

Execute a SOQL query and optionally generate an Explain Plan or use Tooling API.

./sfu query -q "SELECT Id, Name FROM Account LIMIT 10" -p -u user@example.org
sfu query -f query.soql -t -u user@example.org

Options:

  • -q, --query : SOQL query string.
  • -f, --file : Path to a .soql file.
  • -t, --tooling: Use Tooling API.
  • -p, --plan: Generate Explain Plan (Standard API only).

Output: JSON results, execution time, and query_results.csv.

REST API

Make REST API calls to Salesforce endpoints.

./sfu rest -m GET -r /services/data/v60.0/sobjects/Account/describe -u user@example.org
./sfu rest -m POST -r /services/data/v60.0/sobjects/Account -p payload.json -u user@example.org

Options:

  • -m, --method : HTTP method (GET, POST, PATCH, DELETE).
  • -r, --resource : Relative REST API URL.
  • -p, --payload : JSON payload file for POST/PATCH.

Output: JSON response in rest_api_response.json, CSV for GET if applicable.

Bulk API 2.0

Perform bulk operations with CSV input.

./sfu bulk -s Account -o Insert -d data2.csv -u user@example.org
./sfu bulk -s Account -o Upsert -d data.csv -e ExternalId__c -u user@example.org

  • -s, --sobject
